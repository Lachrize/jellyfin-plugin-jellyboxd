# Jellyboxd Sync — Jellyfin plugin

Two-way sync between your **Jellyfin** server and your **Jellyboxd** app, for
**every user** (each on their own account), plus a rating UI inside the Jellyfin
web client.

- **Movies & whole series** → watched, rating and favourite sync **both ways**.
- **Seasons & episodes** → watched sync.
- **Multi-user**: one shared key, every Jellyfin user syncs to their own
  Jellyboxd account (routed by Jellyfin user id) — no per-user setup.
- A **“Ma note” rating widget** under the cast on each detail page, and a
  **rating badge** on poster cards.

## Install (as a Jellyfin plugin repository)

1. Jellyfin → **Dashboard → Plugins → Repositories → +**
   - **Name:** `Jellyboxd`
   - **URL:** `https://github.com/Lachrize/jellyfin-plugin-jellyboxd/raw/main/manifest.json`
2. **Catalog → Jellyboxd Sync → Install**, then **restart Jellyfin**.

## Configure (admin, once)

1. In **Jellyboxd → Paramètres → Synchronisation Jellyfin**, copy the **sync key**
   (it's auto-generated by Jellyboxd; also printed in the Docker logs on start).
2. In **Jellyfin → Dashboard → Plugins → Jellyboxd Sync**:
   - **Jellyboxd URL** — the address of your Jellyboxd app, **reachable from where Jellyfin runs**:
     - Jellyfin **native** (same machine as Jellyboxd) → `http://localhost:3002`.
     - Jellyfin **in Docker / on a NAS** → the **host IP** (`http://192.168.1.50:3002`) or `http://host.docker.internal:3002`. **Never `localhost`** — inside the Jellyfin container that means the container itself (you'll see `Connection refused` in the logs).
   - **Jellyboxd sync key:** paste the key from step 1.
   - **Jellyfin username:** **leave empty** to sync *all* users (fill only to
     restrict to one account).
3. **Save.** Notes / watched / favourites now sync both ways, for every user.
   (Use **Test connection** to confirm the key is accepted.)

> Requires Jellyfin **10.11+** (.NET 9). One plugin install per Jellyfin server
> covers all its users — they don't configure anything.
>
> **Docker-friendly (v1.0.3.0+):** the rating widget is injected into the web
> client *in memory*, so it works even when `/usr/share/jellyfin/web` is
> read-only (containerized Jellyfin running as non-root) — nothing to mount or
> chmod. And **no TMDB key is needed on the Jellyboxd side** for sync to work.

## Troubleshooting

- **`Connection refused` in the Jellyfin logs** → the Jellyboxd URL isn't reachable from the Jellyfin container. Use the host IP or `host.docker.internal:3002`, not `localhost` (see step 2).
- **The note widget doesn't show** → hard-refresh the web page (`Cmd/Ctrl+Shift+R`); the browser caches `index.html`. Check the logs for `[Jellyboxd] … widget … in memory` (Docker) or `Injected the rating widget` (native).
- **You rate in Jellyfin but nothing appears in Jellyboxd** → confirm the movie is *identified* in Jellyfin (has a TMDB/IMDb id on its metadata); the sync needs at least one provider id to match the item.

## Develop

- Needs the **.NET 9 SDK** (Jellyfin 10.11 targets `net9.0`).
- `./build-release.sh` — builds, packages `dist/`, and prints the MD5 to put in `manifest.json`.
- `./deploy.sh` — builds and installs into a local Jellyfin (dev only; stops/starts the app).
